Tutorial i HTML5

Doctype

Doctype skrivs i kod <!DOCTYPE html> och står överst i ett html-dokument för att visa vilken standard som är använd för dokumentet.

Teckenkod

Teckenkod anges i koden <meta charset=" här skriver man in vilken teckenkodning som används på sidan, vanligtvis utf-8 "> detta för att berätta för dator hur denne ska konverterar hemsidans text i bytes när det lagras i en fil, och hur den sen kan konverterar den tillbaka igen när filen ska läsas. <meta charset="utf-8"> anges överst i <head>-elementet om sådant finns annars direkt under Doctype.

Språk

Det anses vara god smak att ange hemsidans huvudsakliga språk, detta för att tex optimera för sökmotorer ska kunna inkludera eller utesluta hemsidor i sökresulta beroende på hemsidans språk. Detta görs genom att lägga till lang=" och de språk som använd, tex sv för svenska eller en för engelska " i <html>-taggen.

Validering

Validering av en hemsida kan enklast göras med W3C validator, på validator.w3.org, genom att validera en hemsida kan upptäcka om det finns kod som inte uppfyller HTML5's rekommenderade standard. Tex kan man upptäcka;

  • obligatoriska element som saknas, tex <title>-elementet
  • att slut-taggar saknas
  • misstag i nästlade listor
  • Taggar som saknar rätt attribut, tex <img> måste ha ett src attribut.
  • element eller innehåll som hamnat på fel ställe, tex <p>-text i <head>-sektionen.

HTML-dokuments uppbyggnad

HTML-dokument är uppbyggt med taggar(<>) dessa använda runt all text som förekommer i ett dokument, ett mycket enkelt HTML-dokument innehåller endast; 

<!DOCTYPE html>
<title>Ett pytte HTML Dokument</title>
<p>Detta är allt som behövs./p>

Standarden på dokumentet(html), title på dokumentet och lite text, allt övrigt är valfritt, dock är det sedvanligt att man också har med ett head och ett boby element, ett html element och om man skriver på svenska är det mycket nödvändigt med ett meta element med ett charset attribut innehållande utf-8 för att få svenska å,ä,ö, då ser det ut på följande vis;

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
	<title>Ett lite större HTML Dokument</title>
</head>
<body>
	<p>Detta är allt som behövs och lite till./p>
</body>
</html>

Struktur

body

Allt innehåll på sidan skrivs i body-taggen, det är innehållet i denna som syns i webbläsaren.

Exempel på body-tagg i kod;

<body>
<h1>Rubrik</h1>
<p>Lite text</p>
</body>

head

Denna tagg innehåller "osynlig" information som används för att ge information till webbläsaren, sökmotorer mm. Detta syns inte på hemsidan. Sidans title, teckenkodning och länkar till stilmallar etc. är exempel på information som anges i head-taggen.

Exempel på head-tagg i kod;

<head>
<meta charset="utf-8>
<title>Sidans title</title>
</head>

html

Denna tag anger att koden och sidan är ett HTML-dokument. Denna tagg omger allt på sidan utom Doctype. I denna tagg kan man även ange sidans språk.

Exempel på html-tagg i kod;

<html lang="sv">
<head>
<title>Sidans title</title>
</head>
<body>
<p>Lite text</p>
</body>
</html>

title

I title-taggen anges sidans titel som syns i fliken överst i webbläsaren, detta element anges i head-taggen.

Exempel på title-tagg i kod;

<title>Sidans titel</title>

Metainformation

meta-taggar med name-attributen

Meta-taggar med namn-attribut används för att ge dold för användaren-vänlig information till webbläsaren och tex sökmotorer. Koden anges i head-elementet.

author

Anger sidans författare, ett exempel kan se ut såhär:

<meta name="author" content="Emily Elmseld">

description

Används för att beskriva hemsidan, ett exempel;

<meta name="description" content"HTML5-tutorial">

generator

Anger ett av de mjukvaruprogram som används för att generera sidan och kan där med inte användas på handskrivan hemsidor, ett exempel;

<meta name="generator" content="FrontPage 4.0">

http-equiv

http-equiv attributet ger en HTTP-huvud information/värdet av attributets innehåll. Http-equiv kan användas för att simulera en HTTP-response tex uppdatera en hemsida var trettionde sekund;

<meta http-equiv="refresh" content="30">

keywords

Används för att ange nyckelord för hemsidan, tex för bättre sökoptimering. Ett exempel;

<meta name="keywords" content="HTML5, tutorial, skoluppgift">

robots

robots-taggen används för att leverera direktiv till sökmotorer, detta genom att skriva en lista som består av särskilda fördefinierade termer och de separeras med kommatecken, som ger instruktioner till sökrobotar etc. Ett exempel som säger till sökrobotar att inte besöka någon del av hemsida;

<meta name="robots" content="nofollow">

Text

blockquote

Ett blockquote element skapar en indragning i texten lika mycket från båda håll, ett exempel i kod;

<blockquote>
<p>If you can dream it you can do it!</p>
<blockquote>

Så här ser det ut;

If you can dream it you can do it!

h1, h2, h3, h4, h5, h6

h1-h6 är element som skapar rubriker de är rankade från 1 till 6 där 1 har högst rank och störst font och 6 lägst rank och minst font, rubrik-elementen är inte gjorde för undertexter, alternativa titlar och taglines som inte är avsedd att vara rubriken för ett nytt avsnitt eller underavsnitt, ett exempel;

<h1>h1 HuvudRubrik<h1>
<h2>h2 Undre huvudRubrik<h2>
<h3>h3 Rubrik<h3>
<h4>h4 Rubrik<h4>
<h5>h5 Rubrik<h5>
<h6>h6 Rubrik<h6>

Så här ser de ut;

h1 HuvudRubrik

h2 Under huvudRubrik

h3 Rubrik

h4 Rubrik

h5 Rubrik
h6 Rubrik

p

p representerar en paragraf och skapar ett nytt stycke för texten inom elementet, mycekt vanligt sätt att skriva text på en hemsida. Ett exempel;

<p>Ett stycke text</p>

Skulle se ut så här;

Ett stycke text

pre

Ett pre element representerar ett stycke förformaterad text, vanliga områden är för att skriva en mailadress och kodstycken som exempeln på denna sida.Ett exempel;

<pre><code><h1>Rubrik</h1></pre></code>

Ser ut så här;

<h1>Rubrik</h1>

span

span taggen används för att gruppera inline-element i ett dokument, själv ger taggen ger ingen visuell förändring. Ett exempel för att färga en del av texten;

<p>Min mamma har <span style="color:green">gröna</span> ögon. </p>

Ser ut så här;

Min mamma har gröna ögon.

Hypertext

a

En a-tagg definierar en hyperlänk, som används för att länka från en sida till en annan.

Relativ och absolut länk

En relativ länk pekar enbart på vart målet är i förhållande till sidan som länken är på och kan bara används när målet är i inom samma domän medans absoluta länkar istället tillhandahåller en specifik adress till länkens mål och funkar därmed även till andra hemsidor. Ett exempel, först på en relativlänk sen på en absolut länk;

<a href="annanSida.html">Relativ Länk</a>
<a href="http://dsv.su.se">Absolut Länk</a>

Så här skulle länkarna se ut;

Relativ Länk Absolut Länk

Länk till ankare

Länk med ankare använd för att länka till ett specifikt mål på en hemsida, tex för att komma till toppen av hemsidan igen eller på en lång sida för att hoppa till ett visst kapitel. Ett exempel på hur de skulle se ut i koden för att länka till toppen på den hemsidan;

<a href="#top">Kom till början av sidan</a>

Detta förutsätter dock att h1-elementet i toppen innehåller ett id-element med värdet top, så här;

<h1 id="top">Huvudrubriken</h1>

Då skulle länken funka så här;

Kom till början av sidan

Bild-länk

Bild länkar används för att länk med tex loggor, koden skulle se ut så här;

<a href="annanSida.html><img src="enBild.png" alt="logga" /></a>

Och skulle fungera så här;

Länk till olika protokoll

Man kan länka till olika protokoll för att tex hämta en fil, exempel på protokoll är ftp och gopher, så här skulle koden för en sån länk se ut för ett ftp- protokoll;

<a href="ftp://[filens adress]">hämta filen här</a>

Den skulle se ut så här;

Hämta filen här

Mailto-länk

En mailto-länk används för att leda en länk till en epostadress istället för en hemsida, i kod ser den ut som följade;

<a href="mailto:emily.elmseld@gmail.com">Maila mig!</a>

Ser ut så här;

Maila mig!

Oordnad, ordnad och beskriven lista

dl

dl-elementet används för att definiera en beskriven lista. Elementet omsluter en eller flera beskrivnings termer, inneslutna i ett dt-element och beskrivning definitioner (definitioner av termerna), inneslutna i ett dd-element.

dt

Ett dt-elementet visar en definition av termerna inom en definitionslista (dl). Ett dt följs vanligen av en eller flera dd element. Flera i rad dt tillskrivs dd element som omedelbart följer gruppen.

dd

Ett dd-elementet representerar beskrivningen, definition, eller värde, en del av en term-beskrivnings grupp i en definitionslista (dl).

Ett exempel på en så lista;

<dl>
 <dt>Kaffe</dt>
 	<dd>svart</dd>
 	<dd>varm</dd>
 <dt>Milk</dt>
 	<dd>vit</dd>
  <dd>kall</dd>
</dl>

Den skulle se ut så här;

Kaffe
svart
varm
Milk
vit
kall

ol

ol-elementet är för att skapa en ordnad lista med listelement(li) de används för att skapa en nummrerad lista, antingen med siffror eller bostäver, standard är med siffror, ett exempel på en ordnad nästlad lista;

<ol>
	<li>Första raden</li>
	<li>Andra raden
		<ol>
			<li>Nästlad ny lista</li>
			<li>Andra raden</li>
		</ol>
	</li>
	<li>Tillbaka i första listan igen</li>
	<li>Sista raden</li>
</ol>

Skulle se ut såhär;

  1. Första raden
  2. Andra raden
    1. Nästlad ny lista
    2. Andra raden
  3. Tillbaka i första listan igen
  4. Sista raden

ul

ul-elementet är för att skapa en oordnad lista med listelement(li) varje ny rad markeras med en punkt, man kan ändra symbol i stilmallen till cirkel, triangel eller punk som är standard, ett exempel på en oordnad nästlad lista;

<ul>
	<li>Första raden</li>
	<li>Andra raden
		<ol>
			<li>Nästlad ny lista</li>
			<li>Andra raden</li>
		</ol>
	</li>
	<li>Tillbaka i första listan igen</li>
	<li>Sista raden</li>
</ul>

Skulle se ut såhär;

  • Första raden
  • Andra raden
    • Nästlad ny lista
    • Andra raden
  • Tillbaka i första listan igen
  • Sista raden

li

li-element är de element som utgör en ordnad och oordnad lista, de kan innehålla nästan vad som helst, koden för ett li-elmenten ser ut som följer;

<li>en rad i en lista</li>

Tabell

table

table-elementet definierar en html-tabell och innehåller minst en av td,th eller tr-elementen för att skapa en tabel.

caption

caption-elementet definerar en tabels rubrik, detta element är inte nödvändigt i en tabel men om de används måste de sitta direkt under table-taggen.

tr

tr-elementet definerar en rad i en html-tabell och innehåller minst ett td och/eller th-element.

th

th-elementet representerar en huvud cell i html-tabell, innehåller header information och texten i denna cell är bold och centerar som standard.

td

td-elementet representerar en standard cell i html-tabell, innehåller data och texten i denna cell är vanlig och vänsterjusterad som standard.

Ett exempel på en tabell som innehåller alla elementen beskriva ovan i kod;

<table>
  <caption>Tabellens Rubrik</caption>
  <tr>
    <th>Första huvud(th) element</th>
    <th>Andra huvud(th) element</th>
  </tr>
  <tr>
    <td>Första standard(td) element</td>
    <td>Andra standard(td) element</td>
  </tr>
</table>

Så här skulle tabellen se ut;

Tabellens Rubrik
Första huvud(th) element Andra huvud(th) element
Första standard(td) element Andra standard(td) element

Utöver den är de vanligt att även använd hight och width attribut för att skapa absoluta värden för höjd och bredd på bilden, ett exempel på kod; 

<img src="enBild.png" alt="logga" height="50" width="30" />

Bildkartor

area

Ett area-element definierar ett område inuti en bild-karta(som är en bild med klickbara områden). area-elementet är alltid inkapslad i en map-tagg.

map

Ett map-element används för att definiera en bild-karta. En bild-karta är en bild med klickbara områden. Map-elementet innehåller ett antal area-element, som definierar de klickbara områden i bildkartan. Måste innehålla ett name-attribut som specifiserar bilden som utgör bild-karta.

Ett exempel på hur detta kan se ut i kod;

<img src="bildMap.jpg" width="400" height="300" alt="ytterklader" usemap="#kladMap">

	<area shape="circle" coords="200,190,52" href="vantar.html" alt="vantar">

	<area shape="circle" coords="309,118,65" href="dregglis.html" alt="dregglis">
</map>
