Tabeller

table-layout

Sätter en tabells layout algoritm, standard är att kolumnens bredd är lika bred som de bredaste obrytbara innehållet i kolumnen men dett kan vara väldigt tidsödslande då hela innehållet i tabellen måste läsas igenom innan man kan sätta layouten, då kan attributet fixed vara bättre då den istället utgår från tabellers bredd och inte innehållet vilket gör att browsern kan börja skriva ut tabellen så fort den får första raden. Ser ut så här i kod:

table-layout: fixed;

border-spacing

Denna property sätter hur stort mellanrummet mellan borderna ska vara, kräver att de inte är kollapsat med föregående kod, ett exempel;

border-spacing: 15px 25px

Första värdet är för horisontellt mellanrum och de andra för vertikalt, om man bara anger ett värdet räknas de som värdet för båda.

Använda CSS med formulär

När man stylar ett formulär är det viktig att formuläret fortfarande är tydligt, vilket sätter vissa begränsningar på stylingen om användaren missar att fylla i en ruta för att den är otydlig kan datan som skickas in med ett formulär bli ofullständig vilket kan bli problematiskt. Eftersom de flesta elementen i ett formulär ligger i input taggen(<inpot>) är det inte särskilt lämpligt att använda denna tag för att stilsätta elementen, och då de är oftast ganska många kan de bli både plottrigt och mycket extra jobb att använda klasser för stilsättning. Då är attribut bästa sättet, då får man med alla input element med samma attribut men lämnar de input element som är något annat, ett exempel kan se ut som följande;

input[type="text"] {
background-color: blue;
}

Backgrunden blir bara blå i de fält som har type=text och inte de som har type=password.
